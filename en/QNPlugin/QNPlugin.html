QNPlugin

Initial entry component of SDK

Main functions:

  • Basic management of Bluetooth devices
  • Verify SDK authorization information
  • System Bluetooth status monitoring, scanning, connection

API

initSdk

The SDK is initialized according to the appId. In the initialization process, the usable permissions are verified according to the appId and configuration file used.

For Android, it is recommended to initialize in 'Application.onCreate'; Suggestions for iOS end in AppDelegate. DidFinishLaunchingWithOptions initialization.

Note: '123456789' appid used in demo is only used to cooperate with demo and debugging, test id has instability and random change. The access party must use the official appId for official use

parameter
name type description
appId String Unique identifier provided by Yolanda
filePath String File path of the configuration file
callback QnResultCallback Callback method that returns the initialized result int See list of status codes for details

setScanListener

Set the scan status monitor, which is used to monitor the action of Bluetooth to start or stop scanning.

parameter
name type description
listener QNScanListener Set scan status monitoring

setSysBleStatusListener

Set the system Bluetooth status monitoring. If the system Bluetooth status changes, this method is called back.

parameter
name type description
listener QNSysBleStatusListener Set the system Bluetooth status monitoring

setLogListener

Set the log output monitor for the APP to record log information output by the SDK.

parameter
name type description
listener QNLogListener Example Set the log output monitor

getSysBleStatus

Obtain the current system Bluetooth status

Return value

Type：QNSysBleStatus

startScan

Start Bluetooth scan

Return value

Type：int See list of status codes for details

stopScan

Stop Bluetooth scan
